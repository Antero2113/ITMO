# Архитектура операционной системы Linux

Linux является многопользовательской многозадачной операционной системой с монолитным ядром. Это означает, что основные подсистемы операционной системы работают внутри единого адресного пространства ядра и взаимодействуют напрямую друг с другом.

Ядро Linux выполняет множество важных функций:

* управление процессами;
* управление оперативной памятью;
* работа с файловыми системами;
* взаимодействие с аппаратурой;
* обработка системных вызовов;
* управление драйверами устройств.

Именно ядро обеспечивает связь между пользовательскими программами и физическим оборудованием компьютера. Пользовательское приложение не обращается напрямую к устройству — все запросы проходят через механизмы ядра Linux.

---

# Пространство пользователя и пространство ядра

Одним из важнейших принципов Linux является разделение памяти и привилегий между пользовательскими программами и ядром системы.

Существует два основных режима работы:

| Режим        | Назначение                           |
| ------------ | ------------------------------------ |
| User Space   | выполнение пользовательских программ |
| Kernel Space | выполнение ядра и драйверов          |

---

## User Space

В пользовательском пространстве работают обычные приложения:

* терминал;
* текстовые редакторы;
* браузеры;
* компиляторы;
* пользовательские процессы.

Эти программы работают с ограниченными правами и не имеют прямого доступа:

* к физической памяти;
* к устройствам;
* к регистрам процессора;
* к памяти ядра.

Такое разделение повышает стабильность системы. Ошибка пользовательской программы обычно приводит только к завершению самого процесса, не затрагивая работу всей операционной системы.

---

## Kernel Space

Kernel Space — привилегированная область памяти, в которой выполняется ядро Linux и драйверы устройств.

Код ядра имеет полный доступ:

* ко всей оперативной памяти;
* к устройствам;
* к системным ресурсам;
* к управлению процессором.

Драйверы устройств работают именно в пространстве ядра, поскольку им необходимо напрямую взаимодействовать с аппаратурой и внутренними механизмами Linux.

Из-за высокого уровня привилегий ошибки в драйверах особенно опасны. Некорректная работа драйвера может привести к зависанию системы, повреждению памяти или аварийному завершению ядра.

---

# Драйверы устройств в Linux

Драйвер устройства — это программный компонент ядра Linux, обеспечивающий взаимодействие между операционной системой и аппаратным устройством либо виртуальным интерфейсом.

Фактически драйвер является посредником между пользовательской программой и оборудованием.

Пользовательская программа выполняет обычные операции:

* открыть файл;
* прочитать данные;
* записать данные.

Ядро Linux перенаправляет эти операции соответствующему драйверу, который уже выполняет необходимые действия.

---

# Типы устройств Linux

Linux поддерживает несколько основных типов устройств.

---

## Символьные устройства

Символьные устройства работают с потоком данных последовательно — посимвольно или побайтно.

Примеры символьных устройств:

* клавиатура;
* COM-порт;
* терминал;
* виртуальные устройства `/dev`.

Для таких устройств характерна последовательная обработка данных без строгой структуры блоков.

В лабораторной работе реализуется именно символьное устройство.

---

## Блочные устройства

Блочные устройства работают с фиксированными блоками данных.

Примеры:

* жесткие диски;
* SSD;
* флеш-накопители.

Такие устройства поддерживают произвольный доступ к данным.

---

## Сетевые устройства

Сетевые устройства используются для передачи пакетов данных через сеть.

Примеры:

* Ethernet-адаптеры;
* Wi-Fi устройства.

---

# Представление устройств в Linux

Одной из ключевых особенностей Linux является концепция:

```text id="e1m4q7"
Everything is a file
```

Практически любое устройство в системе представляется в виде специального файла.

Файлы устройств обычно располагаются в каталоге:

```text id="g5v2z9"
/dev
```

Например:

```text id="r4n8k1"
/dev/tty
/dev/sda
/dev/null
```

После загрузки драйвера лабораторной работы создается файл:

```text id="u6w3d0"
/dev/var1
```

Работа с этим файлом автоматически приводит к вызову функций драйвера.

---

# Системные вызовы Linux

Пользовательские программы не могут напрямую вызывать функции ядра. Для взаимодействия с ядром используются системные вызовы.

Системный вызов — это специальный механизм перехода из пользовательского режима в режим ядра.

---

## Основные системные вызовы

| Системный вызов | Назначение     |
| --------------- | -------------- |
| open            | открытие файла |
| read            | чтение данных  |
| write           | запись данных  |
| close           | закрытие файла |

---

## Связь системных вызовов и драйвера

Когда пользователь выполняет команду:

```bash id="k2t8w5"
cat /dev/var1
```

ядро Linux вызывает функцию чтения драйвера.

При выполнении:

```bash id="d4p6x1"
echo "hello" | sudo tee /dev/var1
```

ядро вызывает функцию записи драйвера.

Таким образом драйвер получает возможность обрабатывать действия пользователя.

---

# Модули ядра Linux

Драйверы Linux часто реализуются в виде модулей ядра.

Модуль ядра — это отдельный программный компонент, который может загружаться и выгружаться без перезагрузки системы.

Файл модуля имеет расширение:

```text id="y0s3m7"
.ko
```

Например:

```text id="m9x4v8"
var1.ko
```

---

## Преимущества модулей

Использование модулей позволяет:

* динамически подключать драйверы;
* уменьшать размер ядра;
* упрощать разработку;
* тестировать драйверы без пересборки системы.

Это особенно важно при разработке и отладке драйверов устройств.

---

# Загрузка и выгрузка модулей

Для подключения модуля к ядру используется команда:

```bash id="q3v6f8"
sudo insmod var1.ko
```

После загрузки модуль начинает выполняться внутри ядра Linux.

Проверить список загруженных модулей можно командой:

```bash id="n7m2d5"
lsmod
```

Удаление модуля выполняется командой:

```bash id="s1w4k9"
sudo rmmod var1
```

---

# Функции инициализации и выгрузки

Каждый модуль ядра содержит функции:

* инициализации;
* завершения работы.

---

## Инициализация

Функция инициализации вызывается при загрузке модуля и выполняет:

* регистрацию устройства;
* выделение памяти;
* создание структур драйвера;
* настройку обработчиков.

Для указания функции используется макрос:

```c id="j8r5x2"
module_init(var1_init);
```

---

## Выгрузка

Функция завершения работы вызывается при удалении модуля и освобождает ресурсы:

* удаляет устройство;
* освобождает память;
* удаляет структуры ядра.

Используется макрос:

```c id="w4n9p6"
module_exit(var1_exit);
```

---

# Символьные устройства и структура cdev

Для регистрации символьного устройства используется структура:

```c id="c5m8t0"
struct cdev
```

Она связывает:

* файл устройства;
* обработчики операций;
* внутренние структуры ядра.

После регистрации ядро Linux начинает перенаправлять операции пользователя в функции драйвера.

---

# Major и Minor номера устройств

Каждое устройство Linux идентифицируется двумя числами:

| Номер | Назначение             |
| ----- | ---------------------- |
| major | идентификатор драйвера |
| minor | конкретное устройство  |

Major number определяет драйвер, который должен обслуживать устройство.

Minor number используется для различения нескольких устройств одного драйвера.

---

# Регистрация устройства

Для регистрации символьного устройства используется функция:

```c id="r8t1k5"
alloc_chrdev_region()
```

Она выделяет major/minor номер и регистрирует устройство в системе.

После этого драйвер становится известен ядру Linux.

---

# Структура file_operations

Основой символьного драйвера является структура:

```c id="z6x2v9"
struct file_operations
```

В этой структуре хранятся указатели на функции, которые ядро вызывает при работе с устройством.

---

## Основные обработчики

| Поле    | Назначение          |
| ------- | ------------------- |
| open    | открытие устройства |
| release | закрытие            |
| read    | чтение              |
| write   | запись              |

Когда пользователь выполняет файловую операцию, ядро автоматически вызывает соответствующий обработчик драйвера.

---

# Обработчик open

Функция `open` вызывается при открытии файла устройства.

Она может использоваться для:

* проверки доступа;
* инициализации данных;
* учета подключений;
* подготовки устройства к работе.

Даже если устройство не требует дополнительной настройки, обработчик обычно присутствует в драйвере.

---

# Обработчик release

Функция `release` вызывается при закрытии файла устройства.

Она используется для:

* освобождения ресурсов;
* завершения операций;
* снятия блокировок.

---

# Обработчик write

Функция `write` отвечает за обработку записи данных в устройство.

При выполнении команды:

```bash id="x7v4n1"
echo "hello" | sudo tee /dev/var1
```

ядро Linux передает данные драйверу через обработчик записи.

В лабораторной работе драйвер определяет количество записанных символов и сохраняет результат во внутреннем массиве.

---

# Подсчет количества символов

Количество переданных данных хранится в параметре:

```c id="d2w7f5"
size_t len
```

Этот параметр содержит число байт, переданных драйверу.

Следует учитывать, что команда `echo` автоматически добавляет символ перевода строки `\n`, поэтому фактическая длина строки увеличивается на один байт.

---

# Динамическое выделение памяти в ядре

Так как количество результатов заранее неизвестно, драйвер использует динамическое выделение памяти.

В пространстве ядра Linux нельзя использовать стандартные функции `malloc` и `free`, поэтому ядро предоставляет собственные механизмы работы с памятью.

---

# kmalloc

Функция:

```c id="f5r8n3"
kmalloc()
```

выделяет память в пространстве ядра.

Она аналогична `malloc`, но предназначена специально для работы внутри ядра Linux.

---

# krealloc

Функция:

```c id="m0x7q2"
krealloc()
```

изменяет размер уже выделенного блока памяти.

При увеличении массива старые данные сохраняются, а память автоматически расширяется.

Это удобно при работе с динамическими структурами данных.

---

# kfree

Функция:

```c id="t9v4s1"
kfree()
```

освобождает память, ранее выделенную в ядре.

Освобождение памяти особенно важно для предотвращения утечек памяти внутри ядра Linux.

---

# Флаг GFP_KERNEL

При выделении памяти используется флаг:

```text id="k6n1p8"
GFP_KERNEL
```

Он обозначает стандартный режим выделения памяти для процессов ядра.

Ядро может временно приостанавливать выполнение потока для поиска свободной памяти.

---

# Обработчик read

Функция `read` отвечает за чтение данных из устройства.

В лабораторной работе обработчик чтения формирует строку со всеми сохраненными результатами и передает ее пользовательскому процессу.

Например:

```text id="p3f7w0"
6
5
7
```

Каждое число соответствует длине строки, ранее записанной в устройство.

---

# Формирование строк в драйвере

Для преобразования чисел в текст используется функция:

```c id="v2m6x4"
sprintf()
```

Она записывает форматированную строку в буфер памяти.

---

# Понятие файлового смещения

В обработчике чтения используется параметр:

```c id="n4q8d1"
loff_t *off
```

Он хранит текущую позицию чтения внутри файла.

Проверка смещения позволяет корректно реализовать окончание файла (EOF), чтобы команда `cat` завершалась после вывода данных.

---

# Передача данных между kernel space и user space

Память пользовательских процессов и память ядра изолированы друг от друга.

Ядро не может напрямую использовать указатели пользовательского процесса, поэтому применяются специальные функции обмена данными.

---

# copy_to_user

Функция:

```c id="s7m3w9"
copy_to_user()
```

копирует данные:

* из пространства ядра;
* в пространство пользователя.

Она используется в обработчике `read`.

---

# copy_from_user

Функция:

```c id="x5f1r4"
copy_from_user()
```

копирует данные:

* из user space;
* в kernel space.

Такая функция часто используется в обработчиках `write`.

---



# Компиляция модулей ядра

Для сборки драйверов Linux используется специальная система сборки ядра.

---

В Makefile указывается:

```makefile id="d6x4p7"
obj-m += var1.o
```

Это сообщает системе сборки, какой модуль необходимо компилировать.

# Использование kernel headers

Компиляция выполняется с использованием заголовочных файлов текущего ядра Linux.

Это необходимо, поскольку драйвер взаимодействует с внутренними структурами ядра.

---

# Виртуальная машина QEMU

В лабораторной работе используется QEMU — программный эмулятор компьютерных систем.

QEMU позволяет запускать Linux для архитектуры RISC-V без физического оборудования.

Эмулятор создает виртуальную аппаратную платформу:

* процессор;
* память;
* устройства;
* сетевой интерфейс.

Это позволяет безопасно разрабатывать и тестировать драйверы устройств.

---

# OpenSBI и U-Boot

При запуске виртуальной машины используются дополнительные программные компоненты.

---

## OpenSBI

OpenSBI выполняет низкоуровневую инициализацию платформы RISC-V и предоставляет интерфейс между аппаратурой и операционной системой.

---

## U-Boot

U-Boot является загрузчиком системы.

Он отвечает за запуск ядра Linux и передачу ему управления.

---

# Общий принцип работы драйвера

Работа драйвера в лабораторной работе происходит следующим образом:

1. Модуль загружается в ядро Linux.
2. Создается символьное устройство `/dev/var1`.
3. Пользователь записывает данные в устройство.
4. Ядро вызывает обработчик `write`.
5. Драйвер вычисляет длину строки и сохраняет результат.
6. Пользователь выполняет чтение устройства.
7. Ядро вызывает обработчик `read`.
8. Драйвер возвращает список сохраненных значений пользователю.
